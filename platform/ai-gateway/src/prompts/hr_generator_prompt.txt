# STANDARD HR ENTITIES (ALWAYS INCLUDE WHEN HR IS ENABLED):

## 1. departments
Required fields:
- `name` (string, required, unique) - Department name
- `location` (string) - Office/building location
- `manager_id` (reference to `employees`) - Department head

## 2. employees
Required fields:
- `first_name` (string, required)
- `last_name` (string, required)
- `email` (string, required, unique)
- `phone` (string)
- `job_title` (string, required)
- `hire_date` (date, required)
- `department_id` (reference to `departments`, required)
- `salary` (decimal) - Only if salary tracking is needed
- `status` (string, options: ["Active", "Terminated", "On Leave"], required)

Set `features.audit_trail: true` for employees.

## 3. leaves
Required fields:
- `employee_id` (reference to `employees`, required)
- `leave_type` (string, options: ["Sick", "Vacation", "Unpaid"], required)
- `start_date` (date, required)
- `end_date` (date, required)
- `reason` (text)
- `status` (string, options: ["Pending", "Approved", "Rejected"], required)

# ENTITY FORMAT (STRICT):
Each entity object MUST contain:
- `slug` (string, plural snake_case like `employees`, `departments`)
- `module` (string) - Always "hr" for HR entities, or "shared" if used by other modules
- `fields` (array of field objects)

Optional entity properties:
- `display_name` (string) - Human-readable name
- `display_field` (string) - Field to show when referenced
- `ui` (object): `{ search, csv_import, csv_export, print }`
- `features` (object): `{ audit_trail: true/false }`

# FIELD FORMAT (STRICT):
Each field MUST contain:
- `name` (string)
- `type` (string): string, text, integer, decimal, boolean, date, datetime, reference

Optional field properties:
- `label` (string)
- `required` (boolean)
- `reference_entity` (string) - For type "reference"
- `options` (array) - For selectable fields (use type: "string" with options, NOT type: "enum")
- `unique` (boolean)

# SYSTEM FIELDS - DO NOT INCLUDE:
- `id`, `created_at`, `updated_at` - These are auto-generated
