# INVENTORY ENTITY PATTERNS:

## Main SKU/Product Entity
The primary entity for tracking stock. Name based on business context (e.g., `products`, `tire_skus`, `dairy_skus`, `items`):
- A human identifier field (`sku`, `code`, `name`) - use as `display_field`
- `on_hand` or `quantity` (integer) - current stock level
- `min_stock` or `reorder_point` (integer) - for low-stock alerts
- Category reference if mentioned
- Price fields if relevant

## Locations Entity (Optional)
If warehouses, bins, racks, or storage locations are mentioned:
- `name` (string, required)
- `code` (string, unique) - location code
- `type` (string, options) - e.g., ["Warehouse", "Bin", "Shelf"]
- `parent_id` (reference to self) - for hierarchical locations

## Stock Movements Entity (Optional)
If receive/adjust/transfer/sell operations or audit trail is needed:
- `product_id` (reference to products entity, required)
- `location_id` (reference to locations, if applicable)
- `movement_type` (string, options: ["Receive", "Adjust", "Transfer", "Sell"])
- `quantity` (integer, required)
- `reference` (string) - PO number, transfer ID, etc.
- `notes` (text)

## Categories Entity (Optional)
If product categorization is mentioned:
- `name` (string, required)
- `description` (text)
- `parent_id` (reference to self) - for hierarchical categories

# ENTITY FORMAT (STRICT):
Each entity object MUST contain:
- `slug` (string, plural snake_case like `products`, `locations`)
- `module` (string) - "inventory" for inventory entities, or "shared" if used by other modules
- `fields` (array of field objects)

Optional entity properties:
- `display_name` (string) - Human-readable name
- `display_field` (string) - Field to show when referenced (e.g., "sku" or "name")
- `ui` (object): `{ search: true, csv_import: true, csv_export: true, print: true }`
- `features` (object): `{ audit_trail: true/false }`
- `inventory_ops` (object) - For quick actions:
  ```
  {
    "enabled": true,
    "quantity_field": "on_hand",
    "receive": { "enabled": true, "label": "Receive" },
    "adjust": { "enabled": true, "label": "Adjust" },
    "sell": { "enabled": true, "label": "Sell" },
    "quick_actions": true
  }
  ```
- `labels` (object) - For printing labels:
  ```
  {
    "enabled": true,
    "fields": ["sku", "name", "price"]
  }
  ```

# FIELD FORMAT (STRICT):
Each field MUST contain:
- `name` (string)
- `type` (string): string, text, integer, decimal, boolean, date, datetime, reference

Optional field properties:
- `label` (string)
- `required` (boolean)
- `reference_entity` (string) - For type "reference"
- `multiple` (boolean) - For multi-select references
- `options` (array) - For selectable fields (use type: "string" with options, NOT type: "enum")
- `unique` (boolean)
- `min`, `max` - For numeric validation

# SYSTEM FIELDS - DO NOT INCLUDE:
- `id`, `created_at`, `updated_at` - These are auto-generated
